# Windows Desktop Control Skill 环境配置指南

本 Skill 依赖 `control.ps1` 脚本实现**高度拟人化的桌面控制**。  
配合 OpenClaw 的多模态视觉能力，可实现自动签到、网页操作、数据录入等日常桌面自动化任务。

## 为什么选择本方案

### 🧬 生物级鼠标轨迹模拟
- **贝塞尔曲线 + 多段随机缓动**：摒弃机械直线，采用三次贝塞尔曲线生成弧线路径；每段移动随机选择缓动指数（2.4~4.2），模拟肌肉力度变化与“顿笔”效果，呈现非匀速的自然节律。
- **分段扫视（视觉追踪滞后）**：长距离（>300px）自动拆分为快速扫视 + 短暂停顿 + 精细校正，复现人类“先快后准”的视觉-运动反馈模式。
- **布朗游走式肌肉震颤**：在轨迹上叠加随机游走噪声并施加阻尼回归力，模拟生理性高频震颤（8-12Hz），末端震颤随接近目标自动衰减，兼顾真实感与精准度。
- **逐帧动态延迟与犹豫停顿**：每步间隔在 5~8ms 随机波动，8% 概率在中段插入 15~40ms 额外停顿（模拟决策犹豫），2% 概率跳过延迟（模拟神经脉冲密集），避免规律性节拍。
- **参数全随机化**：每次移动独立随机化步数（20~50）、抖动幅度、震颤增量、阻尼系数等，消除固定算法指纹，让每次操作都是唯一轨迹。

### 🛡️ 真实环境，零自动化标记
- 脚本通过原生 PowerShell 与系统 API 直接模拟输入，**不涉及任何浏览器驱动或自动化框架**。
- 浏览器启动、鼠标点击均为系统级操作，`navigator.webdriver` 为 `undefined`，所有交互事件 `isTrusted` 属性为 `true`，天然通过核心检测点。
- 无虚拟机痕迹，无第三方依赖，完全运行于真实 Windows 物理环境。

### 🔧 轻量、健壮、易部署
- 所有组合键（Ctrl、Shift）采用 `try/finally` 保证释放，避免卡键。
- 截图操作严格释放 GDI 资源，防止内存泄漏。
- 单文件纯 PowerShell 实现，无需安装任何模块或运行库。
- 截图固定覆盖为 `last_screen.png`，不留垃圾文件。

### 🔄 与 OpenClaw 原生视觉闭环
1. OpenClaw 调用 `snapshot` 获取最新屏幕截图。
2. 利用内置多模态模型识别目标元素（按钮、图标、输入框），直接输出绝对屏幕坐标。
3. 驱动 `control.ps1` 执行点击、打字、滚动等操作。
4. 结合循环判断与自主决策，可实现自动签到、数据抓取、表单填写、网页跳转等复杂工作流。

## 可能的应用场景
- **日常签到**：自动打开网页，识别签到按钮并点击，完成每日打卡。
- **网页操作**：自动填写表单、搜索关键词、翻页滚屏、复制所需内容。
- **数据录入**：读取 Excel 或消息，将内容输入到指定软件。
- **重复性测试**：模拟真实用户操作，对桌面应用或 Web 页面进行回归测试。
- **辅助办公**：为行动不便者提供高可定制化的键鼠动作模拟。

## 1. 确认 control.ps1 存放位置

将 `control.ps1` 放在一个固定的文件夹中（例如 `C:\Users\你的用户名\.openclaw\Workspace`）。  
**记下该文件夹的完整路径**，后续步骤需要使用。

## 2. 将脚本目录加入 PATH（推荐添加至用户环境变量）

### 图形界面方式
1. 按 `Win + R`，输入 `sysdm.cpl` 并回车，打开“系统属性”。
2. 切换到 **高级** 选项卡，点击 **环境变量(N)**。
3. 在 **用户变量** 区域中找到 `Path` 变量，选中后点击 **编辑**。
4. 点击 **新建**，粘贴你的脚本文件夹路径（例如 `C:\Users\你的用户名\.openclaw\Workspace`）。
5. 依次点击 **确定** 保存所有窗口。

### PowerShell 命令行方式（推荐）
以当前用户权限打开 PowerShell，执行以下命令（请替换为你的实际路径）：
```powershell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Users\你的用户名\.openclaw\Workspace", "User")
```
执行完成后，**关闭并重新打开**所有 PowerShell 窗口，使新 PATH 生效。

## 3. 验证环境变量

打开一个新的 PowerShell 窗口，直接输入：
```powershell
control.ps1
```
如果输出类似以下提示信息，表示 PATH 配置成功：
```
请指定 -action 参数: move/click/rightclick/doubleclick/middleclick/wheel/snapshot/type/selectall/copy/paste
```
如果提示“无法将 control.ps1 识别为 cmdlet...”，请检查路径是否正确，并确认已重启终端。

## 4. 配置 Skill 文件

1. 将仓库中提供的 `skill.md` 文件放入 OpenClaw 的 Skill 目录（通常为项目根目录下的 `skills/` 文件夹）。
2. 该 `skill.md` 已默认使用 `control.ps1` 作为命令（无需路径），因为已经将脚本目录加入 `PATH`，所以**无需手动修改 skill.md 中的任何命令路径**。
3. 如果 OpenClaw 运行意外报错“未找到 control.ps1”，请再次确认环境变量是否在当前会话中生效（可运行 `$env:Path` 检查是否包含对应目录）。

## 5. 快速测试

在 OpenClaw 中尝试以下指令，验证 Skill 是否正常工作：
- “截取当前屏幕”
- “移动鼠标到 500,300”
- “在桌面上找到回收站并打开”

如果能成功执行截图、移动和视觉点击，说明整个环境已就绪。  
之后即可尝试更复杂的自动化任务，例如 **“打开浏览器，进入 example.com 并点击签到按钮”**。

## 常见问题

**Q：为什么配置了 PATH 后还是提示找不到脚本？**  
A：请重启所有已打开的 PowerShell 或终端窗口，系统环境变量的变更需要新会话才能加载。亦可尝试重启 OpenClaw 服务。

**Q：我可以将脚本放在其他路径吗？**  
A：可以，只要将该路径加入 `PATH` 即可。如果不想修改 `PATH`，也可以在 `skill.md` 中将所有 `control.ps1` 替换为脚本的绝对路径（如 `"C:\...\control.ps1"`）。

**Q：截图保存在哪里？**  
A：截图固定保存在 `C:\Users\你的用户名\.openclaw\Workspace\last_screen.png`，脚本首次运行时会自动创建目录，且每次截图覆盖同一文件，不产生冗余文件。

**Q：这种模拟会被平台检测出来吗？**  
A：鼠标轨迹采用多段随机缓动 + 布朗震颤 + 动态延迟等生物特征模拟，系统级点击保证 `isTrusted: true`，设备为真实环境，因此极难被静态规则识别。但仍需注意操作频率和规律性，避免长时间高频重复完全相同的序列。

**Q：可以用来做小红书、网银等高风险自动化吗？**  
A：本脚本仅供合法的自动化测试、办公辅助以及个人效率提升使用。请遵守目标平台的服务条款，因违规使用产生的任何后果由使用者承担。
